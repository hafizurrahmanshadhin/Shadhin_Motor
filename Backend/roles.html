Permission Matrix

Use these keys later in Laravel gates, policies, middleware and sidebar Blade partials.

Role Permission Scope Notes Actions
superadmin * Full access. Can create roles and assign permissions.
admin Orders, Samples, Gallery, Reviews, Frontend Content, Media, Employees, Users view/create/edit Can operate content and users without destructive delete access.
subadmin Orders, Samples, Gallery, Reviews, Media, Employees Can operate core shop modules without finance or role control.
manager Orders, Samples, Gallery, Reviews, Frontend Content, Employees, Wages, Finance Operational lead with team, wage and finance visibility.
employee Employees view/edit, Wages view Internal staff view with limited sidebar.
finance Orders view, Finance view/create/edit/export Payment and ledger access only.
editor Samples, Gallery, Reviews, Frontend Content, Media Public website management without finance/users.
user Orders view only Customer or low-access dashboard role for basic order visibility.

Laravel RBAC Map

Keep this static structure ready for database-driven permissions.

Store granular permissions as keys like users.view, users.create, users.edit and users.delete. Sidebar visibility should depend on the module view permission.
For Laravel + React Inertia, send the authenticated user's permissions as shared props, then hide buttons and block routes with Gates or policies.
Suggested tables: roles, permissions, role_permissions, users.role_id and user_permissions for direct allow/deny exceptions on a specific user.
